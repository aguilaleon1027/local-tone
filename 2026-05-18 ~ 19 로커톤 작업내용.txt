[2단계] 데이터베이스 설계

Supabase SQL Editor에서 booking 테이블 생성

CREATE TABLE booking (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL,
  phone TEXT NOT NULL,
  email TEXT,
  booking_date DATE,
  hanbok_id UUID REFERENCES hanbok(id),
  status TEXT DEFAULT '대기',
  created_at TIMESTAMP DEFAULT now()
);

설계 포인트
- hanbok_id는 NULL 허용 (한복 미선택 예약도 가능)
- hanbok_id는 hanbok 테이블 외래키 참조
- status 기본값 '대기'로 예약 접수 상태 추적
- id는 UUID 자동 생성, created_at은 현재 시각 자동 입력

[5단계] 데이터 저장 시퀀스

사용자가 예약 신청 버튼 클릭
  ↓
프론트엔드: 이름, 전화번호 필수값 유효성 검사
  ↓
선택한 한복 목록 순회 (미선택 시 hanbok_id=null로 1건 처리)
  ↓
각 한복별로 POST /api/booking/create 호출 (Promise.all 병렬 처리)
  ↓
백엔드: BookingCreate 모델로 요청 수신 및 유효성 검사
  ↓
백엔드: NULL 필드 제외 후 Supabase booking 테이블에 INSERT
  ↓
Supabase: 자동으로 id(UUID), status('대기'), created_at(현재시각) 생성
  ↓
백엔드: 저장된 레코드 반환
  ↓
프론트엔드: 모든 요청 성공 시 완료 화면 표시
           실패 시 에러 알림 표시

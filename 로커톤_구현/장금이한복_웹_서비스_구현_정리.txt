장금이 한복 웹 서비스 구현 정리

-프로젝트 개요
프레임워크: React + Vite (프론트엔드) / FastAPI + Python (백엔드)
DB: Supabase
배포 형태: 모바일 최적화 단일 페이지 앱 (최대 너비 480px)
주요 외부 API: Kakao Maps JS API, Kakao Local Search API, Resend (이메일), Gemini AI

요구사항 1. AI 한복 피팅
구현 내용

장금이 한복의 한복사진을 저장해서 웹으로 서빙하기 위한 db구축(supabase)
-> 이 과정 아직 장금이 공방 실제 한복 사진을 받지 못한 상태 
-> 현재는 네이버에 있는 한복 사진 3개 정도 가져와서 테스트 형태로 구현한 상태
-> 장금이 공방에서 사진 받을 시 해당 사진으로 db구축 및 ai피팅 튜닝 재진행 예정

3단계 피팅 위저드
단계	내용
1단계 — 사진 업로드	드래그앤드롭 + 클릭 업로드, JPG/PNG/WEBP 최대 10MB, 미리보기 및 삭제, 촬영 팁 카드
2단계 — 한복 선택	캐러셀 형태로 DB 한복 목록 탐색, 좌우 버튼 및 키보드 방향키 지원
3단계 — 결과 확인	Gemini AI로 생성된 피팅 이미지 표시, AI 스타일 추천 코멘트, 다시 피팅 / 예약 상담 버튼
백엔드 처리 흐름

/api/fitting/upload-photo — 사진 서버 업로드, photo_id 반환
/api/fitting/generate — photo_id + hanbok_id 받아 Gemini AI로 피팅 이미지 생성
/api/fitting/result/{id} — 결과 이미지 URL 폴링

요구사항 2. 주변 지역 지도 서비스
구현 내용
카카오맵 연동

Kakao Maps JS SDK 동적 로드 (services + clusterer 라이브러리)
초기 줌 레벨 3, 장금이 한복(행궁로 31-3) 기준 중심 설정
가게 위치에 커스텀 오버레이 핀 고정 표시
카테고리 검색

맛집(FD6) / 카페(CE7) / 관광지(AT4) 3개 카테고리 탭 필터
백엔드에서 Kakao Local Search API 3페이지 동시 요청(asyncio.gather)으로 최대 45개 장소 반환
검색 반경 3km (장안문 포함 행궁동 일대)

마커 클릭 or 리스트 탭 클릭 시 상세 카드 표시
상호명, 카테고리, 도로명/지번 주소, 전화번호, 거리 표시
전화 연결 버튼 + 카카오맵에서 자세히 보기 링크
지도 클릭으로 임의 장소 검색

요구사항 3. 예약 폼
구현 내용
내국인 예약 폼 (한국어)

이름, 연락처, 이메일(선택), 예약 날짜 입력
한복 선택 그리드: 카테고리 필터(전체/여자/아동) + 최대 2개 노출 후 스크롤
복수 한복 동시 선택 가능, 한복별로 예약 건 개별 등록
제출 시 Resend API를 사용해서 설정한 관리자 이메일로 알림 자동 발송

외국인 예약 폼 (다국어)

영어(EN) / 중국어(ZH) / 일본어(JA) 3개 언어 선택 탭
폼 전체 텍스트(placeholder, 라벨, 버튼, 안내문구) 선택 언어로 서빙
한복 카테고리 필터 라벨도 선택 언어로 표시
EN: All / Women / Men / Children
ZH: 全部 / 女性 / 男性 / 儿童
JA: 全て / 女性 / 男性 / 子ども
이름, 이메일, 연락수단(WhatsApp/Instagram/KakaoTalk), 날짜 입력
제출 시 동일하게 관리자 이메일 알림 발송
이메일 알림 형식 -> 장금이 공방 측 요구사항
